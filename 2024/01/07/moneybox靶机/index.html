moneybox靶机

2024-01-07
本文总阅读量次

一、下载好Oracle VM VirtualBox

https://www.oracle.com/cn/virtualization/virtualbox/

二、准备一个moneybox靶机

1.管理->导入虚拟电脑

2.选择MoneyBox.ova，然后点击下一步->完成

3.右击Debian Custum CTF->设置

4.usb设备选择1.1，连接方式选择仅主机，点一下高级，记住mac地址的后四位，设置完成后双击启动

三、打开VMware

1.编辑->虚拟网络编辑器->更改设置

2.将桥接模式设置为Host-Only，然后点击确定，

3.右击kali->设置

4.网络适配器设置为桥接模式，网络适配器2设置为NAT，然后打开kali

四、拿下flag

1.ifdown eth0 关闭eth0网卡

ifup eth0 打开eth0网卡

2.使用ifconfig命令查看eth0的网卡配置状态，看是否在192.168.56.0/24

3.arp-scan -l -I eth0 查看192.168.56.0/24范围内的所存活的ip

4.对照刚刚记住的靶机的mac地址CA4F，可以看出靶机的ip为192.168.56.104

5.nmap -sn 192.168.56.104

6.nmap -p 1-1000 192.168.56.104

7.nmap -p 21,22,80 -sC 192.168.56.104

可以发现一个Anonymous用户名和一个名为trytofind.jpg的文件

8.dirsearch -u “192.168.56.104” -x 403,404,401

扫描出一个blogs目录

9.打开kali自带的火狐浏览器，输入192.168.56.104，右击查看页面源代码，没有发现什么有价值的内容

10.加上刚刚扫描到的blogs目录，发现一个秘密目录

11.进入秘密目录

得到一个密钥，感觉很关键，先放在这里

12.ftp 192.168.56.104

用户名和密码为：Anonymous;Anonymous

13.输入help查看有哪些名可以使用

14.ls查看一下

15.使用get命令下载文件

get trytofind.jpg

16.文件下载后打开发现是一个图片，但貌似获取不到什么有用的信息

17.尝试用二进制的形式查看图片

strings trytofind.jpg

发现有很多括号，正常图片是不会出现这些内容的

18.steghide info trytofind.jpg查看图片是否藏了文件

找不到命令steghide，执行以下apt-get install steghide

可以发现确实隐藏了一个文件，但是好像缺少了一个密钥

19.联想刚刚在页面源代码找到的密钥

steghide extract -sf trytofind.jpg

输入密钥3xtr4ctd4t4

成功剥离出一个data.txt文件，用户名提示为renu，对这个提示进行尝试，看能不能通过字典爆出密码

20.hydra -l renu -P /usr/share/wordlists/rockyou.txt ssh://192.168.56.104

得到ssh登录密码为：987654321

21.ssh renu@192.168.56.104

进行ssh登录输入密码

22.ls查看文件，发现有一个user1.txt的文件，那就cat user1.txt

发现第一个flag，现在离成功不远了

23.ls -all查看所有文件包括隐藏文件，有个history历史记录

cat .bash_history

发现还有一个lily用户

24.ssh登录这个lily用户

25.同样ls查看文件发现一个user2.txt文件，cat user2.txt

得到第二个flag

26.ls -all发现有个历史文件

cat .bash_history

27.发现有个sudo -l命令

那么执行一下看看有什么，发现一个/usr/bin/perl

28.打开https://gtfobins.github.io/

搜索perl，点击perl，找到sudo提权模块

29.输入下面的命令

30.cd /root进行提权

ls -all命令查看所有文件，发现有一个.root.txt文件

31.cat .root.txt

查看这个文件可以发现获取到了最终的一个flag
